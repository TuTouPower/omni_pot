# Omni Pot 设计 / spec / 当前实现差异审查

日期：2026-05-20

> 本文由 `docs/review_claude.md` 与 `docs/review_gpt.md` 合并整理而来。审查依据优先级：
>
> 1. `docs/design/omni-pot/chats/`：聊天记录，视为最高优先级的设计决策记录。
> 2. `docs/design/omni-pot/project/`：设计原型与视觉稿。
> 3. `docs/spec.md`：产品规格与行为约束。
> 4. 当前实现：`src/`、`electron/`、`shared/` 中与前端和窗口行为相关的代码。
>
> 本文只记录差异、冲突、待决断点与建议优先级；不代表所有项都已修改。审查为静态审查，未包含完整运行时验收。

## 总体结论

当前代码已经明显贴近新版设计与 spec：翻译窗口已支持源词卡片、语言检测、结果卡片折叠/展开、TTS/复制/收藏；词典窗口已有可编辑源词卡、语言检测、朗读/复制/收藏/查询；识别与截图翻译已共用 `recognize` 窗口体系；设置页服务管理已拆成多个类别；托盘已有主要入口。

剩余问题主要集中在三类：

1. **代码仍未完全对齐已确认方向**：词典结果卡片结构、服务 tab 文案、托盘词典文案、本地 API 帮助按钮位置、托盘点击行为配置等。
2. **文档之间仍有冲突**：尤其是 `chats` 最终决策、设计原型、`docs/spec.md` 之间在快捷键格式、源文本控件、圆角 token、置顶状态等方面没有完全同步。
3. **需要运行时验证的行为**：窗口尺寸/宽度规则、识别/截图翻译自动重跑、托盘快捷键展示、部分设置页细节仍需实测或 e2e 覆盖。

## 处理原则

- **用户可见 UI 术语**优先跟随 chats 最终决策与 spec 的用户入口章节。
- **设计原型明显滞后的功能**不应倒退实现，例如网络代理、重新识别按钮、Wayland 常驻提示。
- **控件实现细节**若 design 与 spec/代码冲突，应优先保留当前更可靠实现，并在 spec/design 偏差中记录，例如翻译输入区 `textarea` vs `contentEditable`。
- **待决断项**不要偷偷选边，应在 PLAN 或 issue 中保持可追踪。

## 已对齐或无需回退的点

### 翻译窗口

- 源文本区域支持自动增长、最大约 8 行后内部滚动。
- Enter 翻译、Shift+Enter 换行，并规避 IME 组合输入误触发。
- 检测语言显示为“检测为 + 原生语言名”，语言名使用主色强调。
- 源文本动作已包含去除换行、去除空格、朗读、复制、清空、翻译。
- 结果卡片已有拖拽、服务名、loading dots、朗读、复制、收藏、折叠/展开、失败重试。
- 结果卡片初始查询时折叠，结果到达后展开；展开 loading 有轻量动画。
- 翻译 store 已使用 request id 防止旧请求覆盖新结果。
- 欢迎页入口提示已合并，不再拆“划词翻译 / 输入翻译”。

### 词典窗口

- 英文输入走英文词典列表，中文输入走中文词典列表。
- 默认列表已符合 spec：英文 `cambridge_dict + ecdict`，中文 `chinese_dictionary + ecdict`，`free_dictionary` 默认关闭。
- 顶部源词卡片已支持可编辑查询词、语言检测、朗读、复制、收藏、查询。
- 模式标签已使用“词典”。
- 标题栏只有置顶按钮、无固定按钮这一点符合 spec；设计稿中可能出现双按钮应视为原型偏差。

### 设置窗口

- 设置窗口无 pin/lock，右上角是最小化、最大化、关闭。
- 通用页已移除网络代理卡片，符合 spec；设计原型里的代理卡片视为滞后。
- 主题已是三按钮分段控件，不是下拉框。
- 字体区域已改为“文字”，无预览块。
- 主色 5 色与默认配置 sky blue 已进入配置。
- 快捷键页已合并为单一“翻译”入口，绑定/解绑基本符合 spec。
- 历史页工具栏基本是一行布局；仍需确认开关 label 精确为“启用”。
- 备份页保留 WebDAV / 本地，不再有阿里云盘。

### 文字识别 / 截图翻译

- 文字识别与截图翻译共用 `recognize` 窗口，模式为“文字识别 / 截图翻译”。
- 布局为左侧图片卡 + 右侧文本卡；截图翻译模式右侧包含识别文本和翻译文本。
- 标题栏只有置顶按钮、无 Lock，符合 spec。
- 底部操作区顺序基本符合 spec：复制图片、引擎、识别语言、翻译模式下目标语言与右侧文本动作。
- 语言切换、服务切换、翻译目标切换的自动重跑行为仍需运行时验证。

### 托盘

- 托盘弹窗已有翻译、词典、文字识别、截图翻译、剪贴板监听、设置、检查更新、查看日志、重启、退出等入口。
- 前四项显示快捷键。
- 剪贴板监听为 checkbox 行为。
- popup 准备完成后回报尺寸，避免固定宽度。

## 设计稿 / spec / 代码已知偏差

这些偏差不建议按旧设计稿倒退实现：

- **网络代理**：设计原型仍有代理卡片；spec 与代码已移除。
- **翻译输入控件**：设计原型是 `contentEditable`；当前实现使用 `textarea`，属于已知实现豁免。
- **Wayland 快捷键提示**：不应在快捷键页常驻展示。
- **重新识别 / 重新翻译按钮**：设计原型仍有按钮；chats/spec 已要求取消独立按钮，改由切换语言或目标语言自动触发。
- **旧插件入口**：当前不实现 `.potext` 插件市场或外部插件 UI。
- **识别窗口中文 UI**：用户可见文案应避免“OCR”三字母，统一用“文字识别”。

## 需要进一步验证

1. `service_settings.tsx` 中编辑 / 删除按钮是否所有分支都带文字。
2. 托盘快捷键最终展示是否使用 `Control + Alt + T`、`Control+Alt+T` 还是 `Ctrl+Alt+T`。
3. `recognize/index.tsx` 是否完全没有“重新识别 / 重新翻译”按钮。
4. `recognize/index.tsx` 的模式标签是否完全是“文字识别 / 截图翻译”。
5. 识别语言、OCR 服务、翻译目标语言变化时是否自动重新识别或翻译。
6. `history_settings.tsx` 启用开关 label 是否精确为“启用”。
7. 窗口宽度规则：内容是否会撑宽窗口，是否只由用户 resize 改变宽度。
8. 词典、识别、截图翻译窗口的 pin 状态是否会污染翻译窗口设置。
9. 备份恢复入口背后的主进程恢复校验是否覆盖配置、历史、随包 DB。
10. e2e 或手工实测：翻译、词典、识别、截图翻译、设置、托盘的黄金路径与边界路径。

## 建议下一步顺序

1. **先修无需产品决策的小型 UI/文案差异**：服务 tab、托盘“词典”、API `?` 位置、快捷键“词典”、复制图片 tooltip。
2. **再修明确 P0 结构问题**：词典结果卡片同构、词典发音行与朗读按钮。
3. **随后补功能入口**：通用页 `tray_click_event` 配置。
4. **最后处理需要决断的视觉与行为模型**：快捷键格式、圆角策略、CSS 初始主色、独立 pin 状态、`textarea` vs `contentEditable`。
