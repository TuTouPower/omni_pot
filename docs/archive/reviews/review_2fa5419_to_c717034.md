## Code commits

### c717034 — feat(dict,detect): add Chinese dictionary SQLite and cld3-asm language detection {#c717034}

1. **`scripts/build_chinese_dict.ts:32` — silent corruption of non-array JSON.** `load_json` falls back to wrapping the raw file content in `[...]` brackets when `JSON.parse` returns a non-array. If upstream data ever ships as a JSON object or NDJSON, the wrapper produces nonsense and the build silently emits a garbage SQLite. Failing loud beats silent garbage. (confidence 80)
2. **`electron/main.ts:140-155` — `'building'` state broadcast before any window exists.** `BrowserWindow.getAllWindows()` is iterated synchronously inside `app.whenReady`, *before* tray/window creation later in the same handler. The initial transition event is dropped on the floor — any renderer mounted while the build is still in progress relies on `chineseDict:check`, but the `onStateChanged` contract implies live transitions are observable. (confidence 80)

### f5514b7 — feat(tray): add custom light popover {#f5514b7}

3. **`electron/tray/index.ts:215-217` — tray context menu unreachable on Linux.** This commit replaces `tray.setContextMenu(...)` with a custom popover triggered by `tray.on('right-click', ...)`. Per Electron docs, `right-click` does not fire on Linux (the OS owns the tray context menu). With the old `setContextMenu` call removed, Linux users lose all tray menu access (left-click still runs the configured action, but settings/quit are unreachable). Either keep `setContextMenu` as a Linux fallback or gate the change to Win/macOS. (confidence 85)

### cf6e197 — fix(config): simplify service management controls {#cf6e197}

4. **`src/services/lingva.ts:48` — HTTP error responses silently produce empty translations.** The commit drops the `if (!resp.ok) throw …` guard so that "status-zero fulfilled responses" don't fail the test. The cost: a real 5xx returning an empty body now flows through `data.translation` as `undefined`, surfacing as a blank card rather than an error toast. Better: keep the `resp.ok` check and adjust the E2E mock to return `200` instead of `status: 0`. (confidence 85)
