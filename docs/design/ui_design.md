# omni_pot 界面设计需求

> 覆盖 omni_pot 全部窗口的 UI 设计需求。
> 功能与数据流的权威定义见 `docs/spec.md`。
> UI 设计稿原型见 `docs/design/example/`（HTML/JSX/CSS）。

---

## 1. 总体风格

- 样式选定为卡片式。
- 整体界面和所有卡片都使用圆角，卡片圆角为 10。
- 软件整体边角也应保持圆角风格（frameless 窗口）。
- 卡片外部背景使用纯白。
- 卡片内部背景使用带轻微色调的浅色背景，接近当前卡片外部的颜色。
- 各类图标按钮的视觉大小应接近汉字大小，不能明显偏小。

## 2. 主题与主色

- 主色调提供两种：橙色、蓝色。
- 主题提供两种：日间、夜间。
- 最终需要支持四种组合：日间橙色、日间蓝色、夜间橙色、夜间蓝色。
- 橙色和蓝色必须克制使用，本软件是工具软件，不应显得花哨。
- 主色只用于少数关键位置，例如：
    - 置顶按钮（激活态）
    - “检测为英文”里的“英文”两个字
    - 翻译符号
    - 其他少数需要强调的关键状态

## 3. 通用顶部栏

适用于翻译、词典、识别、更新器窗口。

- 置顶按钮放在左上角。
- 顶部栏从左到右依次为：
    1. 置顶按钮
    2. 软件名（带主色圆点 logo）
    3. 当前模式（如“· 翻译”“· 词典”“· 识别”“· 更新”）
- 以上元素整体左对齐。
- 右上角只保留关闭符号。

---

## 4. 翻译窗口

### 4.1 模式与语言显示

- 不再区分“列表式”和“默认”。
- 只保留一个默认翻译模式，删除“列表式”相关入口和展示。
- 中文模式下，语言检测结果应显示为“检测为英文”，不要显示“检测为 EN”。
- 下方语言选择应显示“自动检测”，不要显示“auto”。
- 目标语言只显示“简体中文”，不要显示前缀“ZH”。
- 不需要展示四个引擎相关说明。
- 语言转换区域只展示：自动检测、转换符号、简体中文。
- 这三个元素需要整体居中。
- 中间的转换符号当前太小，需要放大。

### 4.2 输入区域

- 输入翻译文本区域使用小卡片样式框起来。
- 输入文本框至少能展示 8 行内容。
- 输入区域里的按钮必须永久可见，不能因为文本过多被遮挡。
- 文本过多时，只允许输入框内部滚动，输入框旁边需要有滚动条。
- 翻译按钮只保留一个翻译符号，不显示“翻译”文字。
- 翻译符号使用主色。
- 翻译按钮不要有独立突兀的背景，应融入卡片。

### 4.3 翻译结果区域

- 翻译结果使用小卡片样式框起来。
- 每个翻译结果卡片内容不能过矮，至少保证主要内容完整显示。
- 折叠符号当前太小，需要放大到接近汉字大小。
- 语音、复制、收藏、折叠这几个按钮放在同一行。
- 这些操作按钮整体放在单个翻译结果卡片的右上角，顺序为：语音、复制、收藏、折叠。
- 出错时显示红色错误文本，并提供重试按钮（仅出错后显示）。

---

## 5. 词典窗口

- 词典窗口也需要解决内容显示不全的问题。
- 单个词典内容卡片不能太矮，需要保证内容完整展示。
- 词典不需要搜索框。
- 词典不需要换行符号。
- 支持收藏单个词汇，每个词的右上角放置收藏按钮。
- 卡片可包含发音、释义、例句等结构化内容。

---

## 6. OCR 识别窗口

### 6.1 布局

- 第一排为通用顶部栏。
- 第二排改为左右布局：左边展示图片，右边展示识别出来的文本结果。

### 6.2 操作区

第三排操作区需要包含以下元素，统一排列：

1. 选择 OCR 方法（下拉）
2. 识别语言选择（下拉，含“自动检测”项）
3. 重新识别按钮
4. 去除换行按钮
5. 去除空格按钮
6. 复制识别出的文本
7. 导出识别出的文本（格式选择支持 md、txt、docx、doc 等）
8. 翻译按钮

### 6.3 按钮与信息调整

- 去掉图片大小显示、图片类型显示、“已识别多少字”、“用了多久时间”。
- 去除换行和去除空格按钮放在操作区，并和其他操作按钮统一排列。
- 去掉识别时复制图片的按钮。
- “重新识别”不要只用一个符号，应做成和其他大按钮一样并显示文字。
- 导出按钮使用导出符号，不要使用云符号。

---

## 7. 截图窗口

- 全屏覆盖层，无常驻可见控件。
- 屏幕图像渲染为全屏背景，其上覆盖半透明遮罩。
- 鼠标拖拽创建选择矩形：主色描边、四角句柄、选区右下角显示尺寸标签。
- 顶部提示条：拖动选取区域 · Enter 确认 · Esc 取消。
- 主色仅用于选区描边与尺寸标签，遮罩与背景保持中性。

---

## 8. 配置窗口

### 8.1 布局

- 左侧边栏：固定宽度，顶部为 omni_pot logo + 置顶按钮，下方为导航按钮列表，底部显示版本号。
- 右侧内容区：顶部为标题栏（页面名 + 路由提示 + 关闭按钮），下方为可滚动的卡片式内容区。
- 激活的导航项高亮，其图标使用主色。

### 8.2 导航与页面

侧边栏包含 8 个页面：通用、翻译、识别、快捷键、服务、历史、备份、关于。

- **通用**：应用卡片（开机自启 / 检查更新 / 本地 API 端口）、外观卡片（主题 / 界面语言 / 字体 / 字号 / 透明背景 / 开发者模式 / 托盘点击行为）、网络代理卡片。
- **翻译**：语言卡片（源语言 / 目标语言 / 第二语言 / 检测引擎）、行为卡片、窗口卡片。
- **识别**：默认识别语言、自动去除换行、自动复制、失焦关闭、识别后隐藏窗口。
- **快捷键**：五个全局快捷键录入框（划词翻译、输入翻译、OCR 识别、OCR 翻译、划词字典），每个带绑定按钮；附 Wayland 提示卡片。
- **服务**：Tabs 切换翻译 / 识别 / 朗读 / 收藏四类，每类显示可拖拽排序的服务实例列表，底部为添加按钮。
- **历史**：翻译历史列表，支持分页浏览、点击查看与编辑详情、清空。
- **备份**：备份目标选择（WebDAV / 本地）、对应连接配置、备份与恢复操作、最近备份列表。
- **关于**：logo + 版本号、简介、官网 / 文档 / 反馈 / 检查更新链接、诊断信息（日志目录 / 配置目录 / 本机 API 地址）。

### 8.3 卡片与控件

- 内容区以卡片分组，卡片有标题。
- 每个设置项为一行：左侧标签（可带副说明），右侧控件（开关 / 下拉 / 输入框）。
- 控件风格与翻译窗口保持一致的圆角与配色。

---

## 9. 更新器窗口

- 使用通用顶部栏，模式标签为“· 更新”。
- 显示版本对比（当前版本 → 最新版本）、发布日期、包大小。
- 渲染更新日志（Markdown）。
- 下载时显示进度条，进度条使用主色填充。
- 底部操作按钮：“稍后提醒”“立即更新”（下载中禁用）。

---

## 10. 系统托盘

- 托盘菜单为原生上下文菜单。
- 菜单项分组并以分隔线隔开。
- 左键点击行为由配置决定（打开配置 / 打开翻译 / 无操作）。
- 菜单项的具体集合见 `docs/spec.md` 第 21 节。

---

## 11. 状态细节

各窗口需处理以下状态：

- **加载中**：翻译结果卡片显示加载占位（骨架或“翻译中…”）。
- **错误**：结果卡片显示红色错误信息与重试入口；对可定位到配置的错误可提供“打开设置”入口。
- **空状态**：翻译窗口首次启动时显示引导（快捷键提示 + 配置入口）。
- **桌面通知**：`translate_hide_window` 等场景下，后台完成操作后以桌面通知告知。

---

## 12. 重点验收标准

- 翻译窗口只有一个默认模式，不再出现“列表式”。
- 语言显示全部使用中文可读文本：检测为英文、自动检测、简体中文。
- 翻译窗口语言转换区域居中，且只包含“自动检测 → 简体中文”的核心信息。
- 翻译输入区、语言转换区、翻译结果区都采用小卡片样式。
- 输入框和结果卡片高度足够，长文本可以在内部滚动，按钮不会被遮挡。
- 翻译结果卡片右上角集中放置语音、复制、收藏、折叠按钮。
- OCR 窗口第二排为左图右文布局。
- OCR 操作区包含方法选择、识别语言、重新识别、去换行、去空格、复制、导出、翻译。
- OCR 窗口去掉图片尺寸、类型、识别字数、耗时等弱价值信息。
- 词典窗口没有搜索框，支持单词级收藏。
- 配置窗口为左导航 + 右卡片布局，覆盖 8 个页面。
- 截图窗口主色仅用于选区描边与尺寸标签。
- 所有图标按钮大小接近汉字大小。
- 主题支持日橙、日蓝、夜橙、夜蓝四种组合，且颜色使用克制。
- 通用顶部栏左对齐：置顶按钮 → 软件名 → 模式标签；右上角只保留关闭按钮。
