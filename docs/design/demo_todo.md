## 2026-05-20 review 后续偏差

### 1. 文字识别设置页范围

`windows/config.jsx` 中 `PageRecognize` 目前包含设计扩展字段：默认识别引擎、动态识别、默认导出格式、截图后动作、提示条、选区描边颜色。

当前 spec 只要求先实现最小集合：

- 默认识别语言（默认：自动检测）
- 自动去除换行（默认开启）
- 自动复制结果（默认开启）
- 失焦时关闭（默认开启）
- 识别后隐藏窗口（默认开启）

阅读 demo 时不要把上述扩展字段当成本轮必须实现项；如后续要做，需要单独决策和排期。

### 2. 快捷键展示格式

设计稿中仍可能出现 `Ctrl+Alt+T` 这类无空格格式。

当前 spec 要求用户可见快捷键使用简洁清晰、带空格的格式：

- Windows / Linux：`Ctrl + Alt + T`
- macOS：`Cmd + Opt + T`

托盘、快捷键页、欢迎页等展示都应以 spec 为准。

### 3. 用户入口术语

设计稿或旧上传参考中仍可能出现“字典词典”“OCR 识别”“OCR 翻译”等旧文案。

当前 spec 要求：

- 用户入口统一为“词典”，不使用“字典词典”作为按钮、菜单、窗口模式名。
- 中文 UI 使用“文字识别 / 截图翻译”，避免显示“OCR”三字母。

### 4. 圆角规则

设计稿 `styles.css` 中仍有多档圆角 token，部分示例可能使用 12px / 16px 主卡片圆角。

当前 spec 要求：主卡片、主要按钮统一 10px；pill、圆形 icon button、颜色圆点等小元素可保留 pill / 圆形语义。

### 5. 代理设置

如设计稿中仍出现“网络代理”卡片，应视为废弃。当前 spec 与代码目标均为不提供代理设置入口。

### 6. 重新识别 / 重新翻译按钮

如设计稿中仍出现独立“重新识别 / 重新翻译”按钮，应视为滞后。当前 spec 要求切换识别语言或翻译目标语言后自动触发对应流程，不显示独立重跑按钮。

### 7. 窗口置顶状态

设计稿只表达视觉结构，不表达配置持久化模型。当前 spec 要求翻译、词典、文字识别、截图翻译窗口的置顶状态各自独立，互不污染。
