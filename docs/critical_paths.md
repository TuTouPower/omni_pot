# Pot Desktop 关键路径

> 日期: 2026-05-06
> 定义依据: "挂了产品就废了" — 每条路径都满足：用户端到端旅程、不可挂、非功能点

---

## 1. 划词翻译全流程

**用户旅程**: 选中任意文本 → 按快捷键/托盘触发 → 翻译窗口弹出 → 多个服务并行翻译 → 结果正确显示

**为什么是关键路径**: 这是 Pot 的核心价值。用户选中一段文本按快捷键就能看到翻译结果，这是大多数用户每天使用的高频操作。划词获取失败、窗口出不来、服务翻译不出结果，任何一环断了产品就是坏的。

**验证点**:
- 快捷键/托盘触发后选中文本被正确读取
- 翻译窗口出现在光标所在显示器
- 已启用的翻译服务并行调用，结果按实例顺序展示
- 源语言 auto 检测正确，检测到语言===目标语言时回退到第二语言
- 翻译结果为字符串 → 只读文本展示；词典结果 → 发音/释义/例句结构化展示
- 翻译完成写入历史记录（如未禁用）

---

## 2. 输入翻译全流程

**用户旅程**: 按快捷键触发输入翻译 → 翻译窗口弹出（空输入框）→ 用户手动输入文本（含中文 IME）→ 按回车 → 翻译结果展示

**为什么是关键路径**: 输入翻译是划词翻译之外的第二大触发方式。翻译窗口是用户直接交互最多的界面，输入框无法打字、IME 冲突、回车不触发翻译，都会导致用户完全无法使用这个入口。

**验证点**:
- 触发后输入框获得焦点，source area 强制显示（即使 hide_source=true）
- 英文、中文、中英混合输入均正常（IME composing 期间不拦截回车）
- 回车触发翻译，Shift+回车换行
- 动态翻译模式下输入后 1s 自动触发
- 翻译完成后可清空输入框、重新输入、重新翻译

---

## 3. OCR 识别全流程

**用户旅程**: 按快捷键/托盘触发 OCR → 截图覆盖层全屏显示 → 鼠标拖拽选择区域 → 截图裁剪 → OCR 窗口显示识别文字

**为什么是关键路径**: OCR 识别是 Pot 的第二大核心功能。截图覆盖层出不来、选区不工作、OCR 服务调用失败、识别结果不显示，任一环节断裂则整个 OCR 功能不可用。

**验证点**:
- 触发后截图覆盖层全屏、置顶、显示屏幕截图
- 鼠标拖拽绘制选择矩形
- 释放鼠标后截图被裁剪保存
- OCR 窗口打开，加载已配置的识别服务
- 识别服务返回文本正确显示
- 右键点击关闭截图窗口不执行操作
- recognize_delete_newline / recognize_auto_copy 配置生效

---

## 4. OCR 翻译全流程

**用户旅程**: 按快捷键/托盘触发 OCR 翻译 → 截图覆盖层 → 选择区域 → OCR 识别 → 识别文本自动填入翻译窗口 → 翻译结果展示

**为什么是关键路径**: OCR 翻译是截图→翻译的端到端联动，把 OCR 识别和翻译翻译两个核心功能串联起来。这是用户遇到图片中外文时的一站式解决方案，挂了意味着用户只能先 OCR 再手动复制文本去翻译，体验断裂。

**验证点**:
- 截图裁剪后自动触发 OCR → 翻译流水线
- OCR 识别结果作为源文本填入翻译窗口
- 翻译服务对 OCR 文本正确翻译并展示结果
- translate_delete_newline / incremental_translate 配置在此流程中生效
- 翻译完成后自动复制行为（如配置）正确执行

---

## 5. 配置持久化全流程

**用户旅程**: 打开配置窗口 → 修改任意设置（语言/服务/快捷键/代理）→ 关闭应用 → 重新打开 → 配置仍然生效

**为什么是关键路径**: 配置持久化是所有个性化体验的基础。如果用户每次重启都回到默认配置，自定义的 API Key、语言偏好、快捷键全部丢失——对于依赖付费翻译服务的用户，这意味着重启后所有付费服务不可用，产品对他们来说就是坏的。

**验证点**:
- 配置窗口打开，8 个导航 tab 均可切换
- 修改设置后 config.json 立即写入磁盘
- config:changed 广播到所有 renderer 窗口，UI 即时更新
- 重启应用后所有配置值与修改前一致
- 首次启动时根据系统语言自动设置 app_language 和 translate_target_language

## 不是关键路径的功能

以下功能重要但不符合"挂了产品就废了"的标准：

| 功能 | 为什么不是关键路径 |
|------|---------------------|
| TTS 朗读 | 增值功能，没有 TTS 翻译照样用 |
| 收藏（Anki/欧路词典）| 增值功能，与核心翻译流程解耦 |
| 翻译历史记录 | 有用但翻译功能不依赖它 |
| 备份与恢复 | 重要但不在日常使用关键路径上 |
| 自动更新 | 不影响当次使用 |
| 剪贴板监听 | 替代触发方式，不是唯一入口 |
| HTTP API 服务器 | 替代接口，核心功能不依赖它 |
| 插件系统 | 扩展机制，内置服务已覆盖核心需求 |
| 服务拖拽排序 | 便利功能，不影响翻译正确性 |
| 窗口装饰/主题/透明 | 视觉体验，不影响功能 |
| 变量名转换 | 小众功能 |

## E2E 测试对应关系

| 关键路径 | E2E 测试范围 |
|----------|-------------|
| 划词翻译全流程 | 模拟选中文本 → 触发翻译 → 验证结果 DOM |
| 输入翻译全流程 | 翻译窗口打开 → 输入文本(英文/中文/混合) → 回车 → 验证结果 |
| OCR 识别全流程 | 触发截图 → 选择区域 → 验证 OCR 窗口识别文本 |
| OCR 翻译全流程 | 触发 OCR 翻译 → 验证翻译窗口出现翻译结果 |
| 配置持久化全流程 | 修改配置 → 重启 → 验证配置值保留 |

**执行规则**: 每条关键路径必须全跑、每次跑、绿了才合并。
