# omni_pot 设计稿与 Spec 差异报告

> 生成日期：2026-05-28
> 对比来源：`docs/spec.md` vs `docs/design/omni-pot/project/` 设计稿原型
> 用途：向用户报告差异，供用户决定如何处理。**设计稿不自动修改。**

## 严重程度说明

- **高** — 功能缺失、架构不符、可能导致实现偏离核心需求
- **中** — 样式/交互/文案差异，需决策但不影响功能骨架
- **低** — 细节微调，实现时顺手处理即可

## 五、设计系统

### 5.1 顶部栏 Titlebar 组件无置顶/固定按钮 [高] 设计稿

| | 内容 |
|---|---|
| **Spec (§4.3)** | 顶部栏从左到右：**置顶按钮、固定按钮**、显示名 `Omni Pot`、当前模式标签 |
| **设计稿** | `shared.jsx` 中的 `Titlebar` 组件：直接从显示名 `omni_pot` 开始，无置顶/固定按钮 |

**注意**：`translate.jsx` 中的 `TitlebarLeft` 组件**正确实现了**置顶→固定→软件名→模式标签的顺序。但 `shared.jsx` 中还有一个未使用的 `Titlebar` 组件存在歧义，需确认哪个是权威组件。实际翻译/设置/识别/词典/更新器窗口均使用 `TitlebarLeft`，该组件与 spec 一致。

### 5.2 窗口右上角控制按钮未区分 [中] 设计稿

| | 内容 |
|---|---|
| **Spec (§4.3)** | 设置/更新器用最小化/最大化/关闭三件套；其余窗口右上角**只保留关闭按钮** |
| **设计稿** | `shared.jsx` 中 `Titlebar` 组件默认渲染完整三件套，无"仅关闭"变体 |

**注意**：`TitlebarLeft` 组件通过 `chrome="wmctl"` 参数区分了两种模式（`translate.jsx:189-205`），实现正确。`shared.jsx` 中的 `Titlebar` 是冗余组件。

### 5.3 图标按钮颜色状态 [中] 设计稿

| | 内容 |
|---|---|
| **Spec (§4.3)** | 未激活时只显示**主色描边**图标，无背景框；激活时**填充主色**，内部线条反色 |
| **设计稿** | `translate.jsx:149-157` — `TitleBtn` 始终 `color: var(--brand-primary)`；背景始终 `transparent`；通过 SVG 的 `fill` 属性区分激活/未激活 |

**评估**：实现接近 spec 意图（fill 控制填充），但"内部线条反色"效果通过 `var(--bg)` 色模拟（`LockGlyph` 中 keyhole stroke），非严格反色。视觉效果应可接受。

## 建议优先处理

1. **收藏系统**（#1.1, #2.2, #3.1）：设计稿中多处出现"收藏"功能，但 spec 完全未定义。需决定是纳入 spec 还是从设计稿移除。
2. **词典窗口**（#2.1, #2.2, #2.3）：词典窗口设计稿与 spec 差异较多，需集中处理。
3. **服务列表**（#3.2, #3.3）：ecdict/lingva 移除，精简为实际实现的服务；key/标签是否展示需统一决策。
