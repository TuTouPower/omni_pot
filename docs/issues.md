# 已知问题记录

*这里记录了当前项目已知的遗留问题、待办修复或需要追踪的缺陷。已解决的问题归档在 `docs/archive/closed_issues/` 目录下。*

## Windows / 打包实机 smoke

- [ ] **语音朗读（TTS）实机发声验证**：自动化 + PowerShell SAPI 已验证发声链路；仍需在 Windows dist 产物中翻译/词典/识别窗口点击朗读人工确认有声音。
- [ ] **打包后基础路径 smoke**：`npm run dist` 后在 Windows 上验证首次启动、托盘、快捷键、截图、设置窗口、识别窗口。

## 翻译

- [ ] **谷歌翻译当前环境失败**：保留为已知问题，不用 mock 隐藏；需要在网络可达的环境复测，或更换默认免费引擎。

## 窗口尺寸记忆

- [ ] **窗口宽度应按类别记忆**：用户上次设置的窗口宽度应在下次打开时恢复。翻译窗口和词典窗口共享同一份宽度记忆；文字识别窗口和截图翻译窗口共享同一份宽度记忆；设置窗口单独记忆。

## 快捷键设置

- [ ] **快捷键设置不要显示"绑定成功"提示**：绑定成功提示会导致布局对不齐，去掉该提示。
- [ ] **一次只能设置一个快捷键**：如果上一个快捷键正在录入（尚未保存），此时点击另一个快捷键的绑定按钮，应自动取消当前录入，只保留新点击的那个处于录入状态。

## 置顶按钮（固定按钮）

- [ ] **翻译、词典、文字识别、截图翻译窗口均应有置顶按钮**：除设置窗口外，所有主要窗口都应有样式和行为完全一致的置顶按钮。当前存在部分窗口缺失或样式不统一的情况。
- [ ] **置顶按钮图钉符号缺少竖线**：当前实现的图钉图标缺少下方的竖线（stick），需要对照 demo 确认正确的图钉样式并修正。

## 去除换行 / 去除空格符号

- [ ] **去除换行、去除空格图标与 demo 不一致**：当前使用的符号不是 demo 设计稿里的符号，需要对照 demo 确认并替换。

## 截图翻译语言显示

- [ ] **截图翻译目标语言显示错误**：识别出中文后翻译成英文时，底栏语言区域应显示"自动检测 → English"，而非"自动检测 → 简体中文"。
- [ ] **目标语言下拉框不应包含"自动检测"**：目标语言是用户主动选择的翻译目标，下拉框里出现"自动检测"选项没有意义，应移除。
- [ ] **语言下拉框出现两个"自动检测"**：源语言（识别语言）下拉框中"自动检测"选项重复出现，需排查去重逻辑。
- [ ] **截图翻译语言标签需显示具体语言名称**：demo 中文字识别结果卡标题后面应显示识别出的语言，格式为"简体中文的文字识别"或"English的文字识别"；翻译结果卡标题同理，格式为"English 的翻译结果"等。当前缺少语言文字标签。

## 词典服务分流

- [ ] **中文字词应只调用中文词典和 ecdict**：当前中文字词查询会调用全部已启用的词典服务，包括英文词典（Cambridge、Free Dictionary），应按设置中中文词典/英文词典的分组严格分流。
- [ ] **英文字词应只调用 Free Dictionary 和 Cambridge**：同理，英文查询不应调用中文词典。
- [ ] **简单中文字词查询无结果**：如"经济"、"自我"、"佛"等常用字词，中文词典查不出结果，需排查 chinese_dictionary 数据库查询逻辑或数据覆盖范围。
