# 已知问题记录

*这里记录了当前项目已知的遗留问题、待办修复或需要追踪的缺陷。当问题被解决后，相关的记录将被移除。如果积累了大量已解决的问题，或该文件过长，可将其归档到 `docs/archive/closed_issues/` 目录下。*

## 核心功能
- [ ] **谷歌翻译失效**：谷歌翻译一直失败，其他翻译引擎正常。
- [ ] **翻译功能合并（输入/选中）**：“选中翻译”和“输入翻译”这两个功能入口需要合并。逻辑为：如果用户操作时选中了文本，则执行“选中翻译”；如果未选中文本，则唤出正常的“输入翻译”窗口。
- [ ] **中文词典查询失败**：使用中文词典查询单字（如“我”）时，总是提示查询失败。
- [ ] **翻译失败重试功能失效**：翻译失败时出现的红色提示和重试图标，点击后没有触发重新翻译的操作。
- [ ] **语音朗读（TTS）功能失效**：翻译窗口点击语音朗读按钮，没有发音/朗读动作。

## UI / 交互
- [ ] **系统托盘菜单UI缺失与显示不全**：
  - 菜单项之间缺少分隔线，与前端 demo 设计稿不一致。
  - 功能展示不全：缺少“检查更新”、“查看日志”和“退出”按钮。
  - 托盘菜单高度被截断，下方约 1/4 的内容（包括可能存在的退出按钮）被挡住无法显示。
  - 菜单项区域存在大块不合理的空白。
- [ ] **窗口圆角带白色背景瑕疵**：软件窗口应该是圆角的，但当前在窗口角落处能看到圆角外边带有白色背景的直角区域。
- [ ] **固定与置顶功能拆分与联动**：
  - 窗口顶部需要独立展示“固定”和“置顶”两个按钮。
  - **默认失焦关闭**：如果用户没有点击“固定”，当焦点离开窗口时，窗口应自动关闭。
  - **点击置顶**：如果点击“置顶”，则自动激活“固定”状态。
  - **点击固定**：可以只固定窗口（不失焦关闭），但不一定置顶。
- [ ] **输入框动态行数限制**：输入文本框在 8 行以内应完整展示所有内容（随内容撑开），如果内容超出 8 行，则固定在 8 行高度并在框内出现滚动条。
- [ ] **翻译结果卡片折叠与加载动效**：
  - 初始查询时，翻译结果卡片应处于折叠状态，翻译窗口仅展示折叠状态对应的高度。
  - 等待翻译结果时，折叠状态的卡片需要显示“正在翻译”的加载动效。
  - 翻译结果返回后，卡片自动展开，同时窗口高度也自动延长以适应内容。
- [ ] **词典结果卡片折叠与加载动效**：同翻译窗口，词典功能刚开始查询时应折叠并显示加载动效，等结果返回后再展开。
- [ ] **词典卡片内容被遮挡**：选中单词查询词典时，单词下方的词性、读音等信息没有完整展示出来，被卡片底部给盖住了。
- [ ] **翻译窗口垂直拉伸限制缺失**：窗口现在可以上下无限拉伸，拉伸过长会导致底部出现大片空白。期望限制：
  - 最大高度（max-height）：最多拉伸到能完整展示所有的内容卡片即可，不要出现多余空白。
  - 最小高度（min-height）：最矮必须能完整展示出第一个翻译结果卡片。
- [ ] **翻译窗口最小宽度限制不准确**：当前最小宽度不够，期望的最小宽度（min-width）是能完整展示中间的语言切换区域（检测到的语言 -> 转换符号 -> 目标语言）。

## 性能 / 架构
