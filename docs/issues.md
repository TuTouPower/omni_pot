# 已知问题记录

*这里记录了当前项目已知的遗留问题、待办修复或需要追踪的缺陷。当问题被解决后，相关的记录将被移除。如果积累了大量已解决的问题，或该文件过长，可将其归档到 `docs/archive/` 目录下。*

## Windows / 打包实机 smoke

- [ ] **语音朗读（TTS）实机发声验证**：自动化已覆盖朗读 IPC/按钮状态链路，但是否真实出声需要在 Windows 打包产物中手动验证：翻译、词典、识别窗口点击朗读后应能听到声音。
- [ ] **打包后基础路径 smoke**：`npm run dist` 后在 Windows 上验证首次启动、托盘、快捷键、截图、设置窗口、识别窗口。

## 欢迎页

- [ ] **欢迎页窗口大小不匹配内容**：欢迎页窗口无法完整展示所有内容，需要鼠标滚动才能看到完整内容。窗口大小应自动适配内部内容高度。
- [ ] **点击"设置快捷键"后欢迎页未自动关闭**：在欢迎页点击"设置快捷键"后弹出设置窗口，但欢迎页仍留在前台。应自动关闭欢迎页。

## 外部 API / 服务端口

- [ ] **服务器端口供外部脚本调用的 API 未编写，缺少文档**：服务端口对外暴露了，但对应的 API 是否已实现？API 介绍文档在哪里？需编写完整 API 并在 `docs/` 下补充文档。

## 设置 — 服务页面统计

- [ ] **服务页面统计数字随当前选中页面变化**：设置里服务页面上方的统计数据（翻译数量、字典数量等）应各自独立统计，不应随用户当前选中的页面切换而变化。

## 翻译窗口

- [ ] **翻译结果展示方式错误**：翻译窗口应默认展示 3 个翻译结果的折叠卡片，显示"正在翻译…"状态；翻译结果返回后自动展开对应卡片，整个窗口自动变大。当前实现与此不符。
- [ ] **翻译窗口最窄宽度过宽**：最窄情况下应以"自动检测 → 目标语言"转换区域为基准，左侧和右侧仅留极少空白。当前留白过多，视觉效果差。
- [ ] **翻译窗口输入区默认/最大行数应为 8 行汉字**：输入文本区域默认展示 8 行汉字，最大高度也是 8 行汉字。用户重新输入大段文本时窗口应自动跟随变化。
- [ ] **输入长文本时出现两个滚动条**：输入长段文本翻译后，输入区域出现两个滚动条，应只有一个。
- [ ] **翻译窗口最大高度应为 3 个卡片 × 8 行文本**：最大高度以 3 个翻译结果卡片（每个卡片内部 8 行文本）为基准计算。
- [ ] **输入翻译与划词翻译应合并为一个功能**：快捷键触发时若用户已选中文本则直接翻译该文本；若未选中文本则弹出窗口等待用户输入。两个功能合并为一个入口。

## 语言名称显示

- [ ] **所有语言名称应使用对应语言的本地名称，禁止使用英文**：翻译窗口、OCR 界面等所有位置的语言名称均应显示为对应语言的本地名称（如"中文"、"English"、"日本語"），不能全部用英文。此问题之前已提过但仍未修复。

## 置顶 / 固定设置

- [ ] **置顶和固定设置应为窗口级别，关闭后重置**：用户设置的"是否置顶""是否固定"应仅对当前窗口生效。窗口关闭后，新打开的窗口应恢复默认（不置顶、不固定）。包括翻译窗口和 OCR 窗口。

## 词典

- [ ] **词典卡片内容展示不全**：词典界面卡片内容无法完整显示，应根据内容自动计算卡片高度。
- [ ] **划词字典改名为"查询字典"或"词典"**：功能入口名称需更新。
- [ ] **中文词典搜索失败**：随便搜一个中文词都搜不出来。
- [ ] **CC-CEDICT 词典持续失败**：该词典查询一直报错。
- [ ] **Free Dictionary 内容极少**：查询结果内容非常少，质量差。
- [ ] **搜索 "Australia" 无法命中**：英文词条搜索也不可用。
