# 已知问题记录

*这里记录了当前项目已知的遗留问题、待办修复或需要追踪的缺陷。当问题被解决后，相关的记录将被移除。如果积累了大量已解决的问题，或该文件过长，可将其归档到 `docs/archive/` 目录下。*

## Windows / 打包实机 smoke

- [ ] **语音朗读（TTS）实机发声验证**：自动化 + PowerShell SAPI 已验证发声链路；仍需在 Windows dist 产物中翻译/词典/识别窗口点击朗读人工确认有声音。
- [ ] **打包后基础路径 smoke**：`npm run dist` 后在 Windows 上验证首次启动、托盘、快捷键、截图、设置窗口、识别窗口。

## 翻译

- [ ] **谷歌翻译当前环境失败**：保留为已知问题，不用 mock 隐藏；需要在网络可达的环境复测，或更换默认免费引擎。
- [ ] **输入翻译与划词翻译入口仍未完全合并**：按设计稿，这两个能力应合并为同一个翻译入口；不要在欢迎页、设置或快捷键配置里继续拆成两个用户入口。
- [ ] **输入中文长句被误判为日语**：例如输入“我爱你”重复长文本时，语言检测应识别为简体中文，不应检测为日语。

## 欢迎页

- [ ] **欢迎页点击跳过后应直接关闭窗口**：用户跳过欢迎页后不要打开/保留翻译窗口，应直接关闭当前窗口。
- [ ] **欢迎页/快捷键提示应按平台显示修饰键**：Windows 显示 Control，macOS 显示 Command，不要把 `CommandOrControl` 原样展示给用户。

## 托盘

- [ ] **系统托盘快捷键提示应按平台显示修饰键**：Windows 显示 Control，macOS 显示 Command，不要展示 `CommandOrControl`。

## 窗口行为

- [ ] **非固定窗口失焦后应自动关闭**：除设置窗口外，其他窗口如果用户没有点击固定，应在失去焦点后自动关闭。

## 翻译窗口 UI

- [ ] **删除空格图标不符合前端 demo**：删除空格按钮应使用设计稿里的符号，不要使用自造图标。
- [ ] **去除换行图标需要按前端 demo 确认**：不要自造图标；后续修复前先对照 demo 确认设计稿使用的具体符号。
- [ ] **删除空格等文案出现原始翻译 key**：例如显示 `Delete_spaces`；需要系统检查是否还有类似未正确翻译出来的 key。
- [ ] **词典入口命名不符合设计稿**：demo 中叫“字典/词典”，不要叫“查询字典”。

## 文字识别 / 截图翻译

- [ ] **文字识别点击翻译与截图翻译应进入同一个截图翻译窗口**：这两个入口对应同一个窗口，统一叫“截图翻译窗口”，不要拆成两个不同窗口概念。
- [ ] **截图翻译窗口布局不符合 demo**：窗口应为左侧一个大的原图卡片，右侧纵向两个小卡片，分别显示文字识别内容和文字翻译内容；具体细节修复前先看 demo。
- [ ] **切换识别语言后应自动重新识别**：文字识别窗口中用户切换语言后，不需要额外点击，应自动重新识别。
- [ ] **识别窗口语言区域不应显示网络图标**：语言项前不要加网络符号，按 demo 移除该图标。
